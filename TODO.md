### 1. Система «имя + обозначение» (tag)

У каждого элемента — два поля:
- **Имя** (name): свободный текст, может дублироваться. Например, «Сервер», «Кондиционер».
- **Обозначение** (tag / designation): уникальный короткий идентификатор, НИКОГДА не повторяется в рамках одной схемы. Типовые префиксы по типу элемента:

| Тип | Префикс | Пример |
|---|---|---|
| Источник (ввод ТП) | `TR` | TR1, TR2 |
| Генератор | `GS` | GS1, GS2 |
| Щит | `PNL` или `ЩС` | PNL1, PNL2 |
| ИБП | `UPS` | UPS1, UPS2 |
| Потребитель | `L` или `C` | L1, L2, L3 |

- Автогенерация при создании: берётся наименьший свободный номер с префиксом типа
- Проверка уникальности при ручном редактировании и импорте/склейке
- При копировании элементов (Ctrl+V) — имя сохраняется, tag получает следующий свободный номер
- Показывается на плашке узла мелким шрифтом под именем: `TR1 · Ввод ТП`

### 2. Связи: расширение

Сейчас связь возможна только выход → вход. Расширить:
- **Выход → выход** (например, два параллельных щита запитываются от одной сборной шины)
- **Вход → вход** (перемычки)
- Переключение конца связи: рукоятка работает и на конце «from», не только «to»
- Тип связи определяется автоматически по тому, какие порты соединены
- Расчёт: для перемычек вход-вход нагрузка распределяется между соединёнными входами как у параллельных входов одного узла

### 3. Три типа линий кабеля

- **Красная сплошная** — напряжение есть и ПЕРЕДАЁТСЯ дальше (активный путь до потребителя)
- **Зелёная сплошная** — напряжение есть, но дальше НЕ ПЕРЕДАЁТСЯ:
  - вход не выбран приоритетом
  - вход отключён вручную (ручной режим щита)
  - поломка где-то ниже по цепи
- **Серая пунктирная** — напряжения нет вовсе (upstream обесточен)

Цвета стрелок и подписей kW подстраиваются под цвет линии. Подпись kW рисуется только на красных (активных) линиях.

### 4. Лампочки на входах — новая логика

На входах узлов с >1 входа:
- **🟢 Зелёная лампочка** — когда к входу подходит **красная** линия (активно, питает)
- **🔴 Красная лампочка** — когда к входу подходит **зелёная** линия (есть напряжение, но не используется — АВР выбрал другой вход)
- **нет лампочки** — когда линия серая пунктирная (нет напряжения)

### 5. Ручной / автоматический переключатель щита

Новое поле у щита: `switchMode: 'auto' | 'manual'`.
- **`auto`** (по умолчанию) — как сейчас: АВР выбирает активный вход по приоритетам, равные параллелят нагрузку
- **`manual`** — пользователь явно указывает активный вход через поле `manualActiveInput: number`. В этом режиме **ни приоритеты, ни наличие напряжения** не влияют на выбор: если пользователь сказал «вход 2» — работает только он. Если на этом входе нет напряжения — щит обесточен, даже если на других есть

В инспекторе переключатель `auto / manual` + в режиме `manual` появляется выбор активного входа (радиокнопки по номеру входа).

### 6. Группы потребителей

Вместо одного потребителя можно добавить **группу** из N потребителей:

- Новый тип узла `consumerGroup` (или поле `isGroup: true` у существующего `consumer`)
- Поля:
  - `count` — число потребителей в группе (1 = обычный потребитель)
  - `items: Array<{ name, demandKw }>` — массив элементов, каждый с СВОЕЙ регулируемой мощностью
  - Либо упрощённый режим «все одинаковые»: одно общее `demandKw` + `count` (UI-переключатель «единообразная / индивидуальная»)
- Инспектор для группы:
  - В режиме «единообразная»: одно поле «Мощность каждого, kW» + поле «Количество»
  - В режиме «индивидуальная»: таблица с N строк «имя + kW», кнопки ➕ / ➖ для добавления/удаления
- Визуализация на холсте:
  - Плашка узла чуть шире и показывает «`L1` · Сервера · **10 × 8 kW** = 80 kW»
  - Либо если мощности разные: «`L1` · Сборка · **Σ 57 kW** · 8 шт»
- **Подпись на подходящей линии** НЕ суммирует, а показывает **произведение**:
  - Уникальные: `10 × 8.0 kW` (если все одинаковые)
  - Смешанные: `57.0 kW (8 шт)` — сумма + количество
- Расчёт: demand группы = Σ(items.demandKw) — так же, как обычный потребитель с суммарной мощностью, но отображение другое
- При копировании/дублировании (Ctrl+D) группа копируется со всеми элементами

### 7. ИБП — байпасный вход

Добавить `bypassInputs: number` (0 или 1). Байпасный вход:
- Отдельный порт, обычно нижний приоритет (или «последний шанс»)
- При работе через байпас: поток идёт **мимо** инвертора, КПД = 100%, батарея не используется, нет зарядного тока
- Активируется только если основные входы обесточены И батарея разряжена (или вручную через флажок «форсировать байпас»)
- В инспекторе отдельная секция «Байпас»
- На плашке: индикатор режима — нормальный / батарея / байпас

## Приоритет 1 — реальная польза, небольшой объём

### Симуляция по времени (разряд батареи)
Кнопка «▶ Разрядить» играет схему во времени:
- Батареи ИБП реально расходуют `kWh` (shared state, тикает ~1 секунда = 1 минута модельного времени)
- Заряжаются обратно при восстановлении сети
- Переключение АВР показывается с задержкой (например, 20 мс → визуально мгновенно, 5 с при возврате)
- Таймлайн аварии: `[0:00 норма] → [0:05 Ввод 1 отказал] → [0:05.02 АВР → Ввод 2] → [...]`
- Можно поставить на паузу, прокрутить, сбросить к началу

### Коэффициент одновременности Кс / Ки
- На потребителе: `kUse` (0.0–1.0), по умолчанию 1.0. Эффективное потребление = `demandKw × kUse`
- На щите: `kSim` (0.0–1.0), по умолчанию 1.0. Суммарная проходная нагрузка щита умножается на Ксим перед передачей upstream
- Показывается в инспекторе отдельной строкой: «Номинал 100 kW · Ки 0.85 · расчётная 85 kW»
- Соответствует ПУЭ / СП 256

### Токи и сечения кабелей
У связи появляется поле `voltage` и `cosPhi`. Глобально у схемы — значения по умолчанию (0.4 кВ / 3ф / cos φ 0.92).
- Расчёт тока: `I = P / (√3 · U · cos φ)` для 3-фазных, `I = P / (U · cos φ)` для 1-фазных
- Подбор минимального сечения жил по таблице ПУЭ (есть стандартная таблица I_long для медных ПВХ-кабелей: 1.5/2.5/4/6/10/16/25/35/50/70/95/120/150/185/240 мм²)
- На линии показывается: `12.5 kW · 19 А · 4 мм²`

### Автоматы защиты на выходах щитов
- У каждого выхода щита поле `In` (номинальный ток автомата, А)
- Стандартный ряд: 10/16/20/25/32/40/50/63/80/100/125/160/200/250 А
- Если расчётный ток > In → линия становится мигающей красной, в инспекторе выводится предупреждение «автомат не тянет»
- Кнопка «Подобрать автоматы» автоматически выставляет ближайший больший стандарт

### Проверки схемы (валидация)
Блок «Проверки» в правом инспекторе при пустом выделении:
- Висящие потребители (нет входящих связей)
- Щиты без выходов / без входов
- Генератор без флага «резервный»
- ИБП с нулевой батареей
- Кабели с превышением тока
- Кольца (уже блокируются при создании, но проверяется при импорте)

Клик по пункту проверки → выделяет проблемный элемент.
