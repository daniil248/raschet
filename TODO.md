## ✨ UX / полировка

- **ups-config: wizard «Настройка ИБП с CRF» (для внешнего вызова из других модулей)** — реализовать в `ups-config/` мастер, который проводит юзера через все этапы (тип, мощность, входы, байпас, K_рез по типам нагрузки, max load factor, parallel, …) аналогично тому что сейчас в `js/engine/inspector/ups.js`. Если запущен из `?from=schematic` (или другого модуля) — все известные параметры (capacityKw, voltage, manufacturer, и т.п.) предзаполнены из исходного узла, проходить нужно только недостающие. Возврат — обновляет узел в исходной схеме (через postMessage / localStorage handshake / URL callback). См. v0.59.624–632 для текущей логики.

- **Расширить колонки и bulk-edit во всех таблицах**: subtype, inrush, breakerMargin, curveHint, voltage, hasN, hasG, weight, dimensions для consumers; больше полей для cable/equipment. Сделать generic registry колонок (декларативный с metadata), чтобы добавлять колонки одной строкой.

- **Применить presets-систему (`shared/table-presets.js`) к таблицам в других модулях**: scs-config (contents, racks), scs-design (links, bom), inventory, logistics, etc. Сейчас работает только в Конструкторе схем (consumers/cable/equipment).

- **Полноценное окно «Справочники / база данных»** — частично сделано в v0.59.641: добавлен переключатель «⤢ На весь экран» (drawer расширяется до 95vw, состояние запоминается в localStorage). Осталось: боковая навигация по типам справочников (ИБП / трансформаторы / кабели / автоматы / АКБ / стойки / PDU / ...) и единая широкая таблица справа с фильтрами/сортировкой и preset-системой (`shared/table-presets.js`). Управление: «Мастер подбора», «+ Добавить вручную», «Импорт XLSX», «Шаблон XLSX». Каждый справочник — это таблица того же класса, что consumers/cable/equipment.

- **Глобальные настройки платформы — переработать в полноценное окно с разделами** (юзер: «давай настройки напряжений уберём только туда, где эти напряжения нужны, либо сделаем нормальное окно, в котором будут разделы и разные настройки программы, а не только напряжения»). Сейчас «Глобальные настройки платформы» содержат только Справочник уровней напряжения. Варианты:
   - (А) **Распределить** настройки по модулям — voltageLevels хранить там, где они реально используются (`engine` / `cable` / `inspector`); глобальную модалку убрать или оставить только под рабочую конфигурацию платформы.
   - (Б) **Расширить модалку**: разделы слева (Напряжения / Параметры по умолчанию / Кабели / Автоматы / Кодировка обозначений / Учётные данные / Локализация / Юниты / Цвета зон / Тёмная тема / ...), справа — содержимое выбранного раздела. Модалка `modal-wide` или fullscreen, scroll внутри.
   - Нужно решение пользователя — какой вариант предпочтительней.
