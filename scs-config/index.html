Конфигуратор СКС / телеком оборудования в разработке

Модуль описывает содержимое стойки — сетевые устройства, патч-панели, серверы — с привязкой к разъёмам PDU, заданным в Конфигураторе стойки.

Что будет в модуле

  • Коммутатор: порты (8/16/24/48), Base-T / SFP / SFP+, класс PoE (off/af/at/bt), потребляемая мощность.
  • Патч-панель: порты (12/24/48), keystone / модульная, категория (5e/6/6A/7/оптика), высота U.
  • Сервер: 1U/2U/4U, мощность Вт, блоки питания (1+1 / 2+0), входные разъёмы (C13/C14/C19/C20).
  • Базовые устройства: КВМ, монитор 1U, ИБП rack-mount.

Интеграция со стойкой

  • Drag-n-drop в карту юнитов стойки: одна единица = N U; нельзя наложить на заглушку или другое устройство.
  • Каждое устройство привязывается к конкретной розетке PDU из конфигурации стойки. Один слот = одно устройство. Проверка перегруза PDU.
  • Параметры хранятся в rack.contents[] — round-trip через serialize/deserialize.

СКС-матрица

  • Для патч-панелей — список соединений «порт ↔ порт» (другие панели / коммутаторы).
  • BOM по патч-кордам: длина × тип × цвет.

Шаблоны

  • Конфигурация (стойка + содержимое) сохраняется как шаблон.
  • Одним кликом применяется к другим идентичным стойкам проекта.

Статус

Модуль создан как stub. Реализация — подфаза 1.24 в ROADMAP.md (задачи 1.24.1–1.24.8). Запускается после Конфигуратора стойки (1.23).
