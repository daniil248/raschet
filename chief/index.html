🧐 ГИП — Главный инженер проекта

Сводный view всех дисциплин проекта в одном месте. Read-mostly, кросс-дисциплинарный — для согласования разделов и проверки целостности проекта.

Что будет в модуле

📊 Сводка по проекту

  • Тип объекта (ЦОД / завод / насосная / офис / своё) и активные дисциплины
  • Стадия проекта, дата ревизии, версии разделов
  • Команда: участники, роли (electrician, mechanic, technologist, ...)

🗂 Перечень оборудования (cross-discipline)

  • Все единицы оборудования проекта со всеми возможными портами по системам (⚡ электрика, 🌊 трубы, 💨 ОВК, 📊 данные, 🔥 пожарка, 🔐 СКУД)
  • Привязка модели / каталог / артикул (или явный «не выбрано»)
  • Расширенный Перечень из Технолога объекта + объединение с Конструктором схем

🔌 Корректность схем

  • Электрические схемы: висячие связи, нет источника, перегруз, недостаток автоматов
  • Механические схемы (после появления модуля механика)
  • СКС-связи: дубли портов, орфаны

🧮 Проверка расчётов

  • Электрика: ΔU в норме, токи, селективность защит
  • Cooling: производительность ≥ IT-нагрузки + резерв
  • UPS: ёмкость ≥ IT-нагрузки + autonomy
  • ТП/ДГУ: запас по ТУ ≥ потребление
  • Cross-discipline: общая мощность ⇄ TP ⇄ DGU

✓ Согласование разделов

  • Кнопка «✓ Согласовано» на каждом разделе с timestamp / подписью
  • История ревизий разделов
  • Замечания / комментарии к разделам

Архитектура

Модуль ГИП — отдельный URL /gip/, аналогично tech-workspace. Все данные проекта (схема, концепция, BOM, отчёты) уже хранятся в project-storage / Firestore — модуль ГИП их read-only агрегирует.

Не дублирует функционал других модулей. Технолог объекта остаётся для своей работы (предпроектная концепция); электрик / механик — в своих Конструкторах. ГИП = сводный читатель + согласователь.

Связанные модули

  • 🏗 Технолог объекта — концепция объекта (ЦОД / завод / насосная / офис / своё)
  • ⚡ Конструктор электрических схем — детальное проектирование электрики
  • 📁 Проекты — управление проектами
  • 📚 Каталог — справочные данные оборудования

Модуль будет реализован в Phase 47 (см. ROADMAP). До этого все сводные view доступны в Технологе объекта (Перечень оборудования) и Конструкторе электрических схем (Перечень электротехнического оборудования, Ctrl+Shift+E).
