🧰 Мастер меж-шкафных связей

Выберите несколько стоек проекта — они появятся рядом для проектирования связей между конкретными портами/патч-панелями. Содержимое шкафов читается из Компоновщика шкафа (LS scs-config.contents.v1). Внутренние патчкорды одного шкафа настраиваются там же — этот модуль про связи между разными шкафами.

Выбранные стойки

Кликните на устройство в одной стойке — оно подсветится. Затем кликните на устройство в другой стойке — появится связь в таблице ниже. Повторный клик по подсвеченному = отмена. Связи внутри одного шкафа — в Компоновщике шкафа.

Меж-шкафные связи

Кабельный журнал (BOM СКС)

Агрегация по типу кабеля: количество линий + суммарная длина с коэффициентом запаса 1.3 (манхэттен × запас). Линии без указанной длины в сумму не попадают, но считаются.

🗺️ План зала

Top-view расстановка стоек на сетке. Перетаскивайте чипы из палитры неразмещённых стоек на план, двигайте их мышью. Масштаб сетки задаётся ниже и используется для подсказки длины меж-шкафных связей (манхэттен-расстояние × масштаб × коэффициент трассы).

🗄 Стойки проекта

Сводка по каждой стойке: занятость U, разбивка устройств по роли, мощность и число меж-шкафных связей (участвует как endpoint). Кнопка «Выбрать» добавляет стойку в мастер связей.

Импорт / экспорт проекта СКС

Backup и шаринг. JSON содержит выбор стоек, все меж-шкафные связи и расстановку на плане зала. Содержимое шкафов и сами стойки (rack-config / scs-config) — отдельные хранилища, в этот файл не входят.

О модуле «Проектирование СКС»

Модуль отвечает за проектирование структурированной кабельной системы объекта: трассы, кроссы, пары порт ↔ порт на уровне здания и этажей, маркировка, длины кабелей, BOM СКС. Roadmap: Подфаза 1.26 (поглотила 1.25).

Где что живёт

  • Конфигуратор шкафа — корпус — типовое железо шкафа (шаблоны).
  • Компоновщик шкафа — наполнение конкретного экземпляра, патчкорды внутри одного шкафа.
  • Реестр IT-оборудования — экземпляры сервер/свич/хранилище с S/N, IP, MAC.
  • Здесь — связи между разными шкафами, трассы, кабельный журнал.
  • Реестр оборудования объекта — всё не-IT имущество площадки.
