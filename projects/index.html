📁 Проекты Raschet

Проект = единица проектной работы по объекту. Содержит схему электроснабжения, СКС (меж-шкафные связи, план зала), шкафы с содержимым, реестры оборудования, заметки. Конфигураторы (корпус шкафа, РУ СН, ИБП и т.п.) — это библиотека шаблонов, они хранят только сохранённые пользователем конфигурации и не лезут в данные проекта.

Список проектов

Что содержит проект

  • Электрическая схема — узлы, линии, кабели, защита (Конструктор схем).
  • СКС — меж-шкафные связи, план зала, кабельный журнал (Проектирование СКС).
  • Шкафы — содержимое каждого экземпляра (Компоновщик шкафа).
  • IT-реестр — экземпляры серверов/свичей с S/N, IP, MAC (Реестр IT).
  • Не-IT имущество — мебель, стеллажи, ЗИП (Реестр оборудования объекта).
  • Заметки проекта — клиент, адрес, сроки, контакты.

Что НЕ хранится в проекте

  • Библиотека корпусов шкафов (Конфигуратор шкафа) — общая.
  • Сохранённые конфигурации РУ СН (mv-config) — общая библиотека.
  • Каталоги элементов, кабелей, автоматов — общие для всех проектов.
  • Шаблоны ИБП, ПДУ, трансформаторов, МДЦ — общая библиотека.

Правило: всё, что проектируется для конкретного объекта — живёт в проекте. Всё, что используется как типовое решение повторно — в конфигураторах.

🔌 Финальная Фаза X — «Управление объектом» как отдельный продукт

Отдельный большой этап в конце roadmap: модуль «Управление объектом» (для действующих, уже построенных объектов — мониторинг, эксплуатация, ЗИП, интеграция с системами мониторинга SNMP / BACnet / Modbus / OPC UA) проектируется как отдельно-деплоимый продукт. Его можно продать клиенту или разместить у него на сервере, с независимыми обновлениями. Уже сейчас архитектура платформы закладывается так, чтобы это стало возможным без рефакторинга:

  • Контракт данных — schema-versioned JSON для всех типов объектов.
  • Транспорт — адаптер shared/project-storage.js: сегодня LS, потом HTTP/IndexedDB без правок в модулях.
  • Модульная обновляемость — manifest.json с semver модуля и требуемой версией schema проекта.
  • Лицензирование — организационный ключ + срок, verify offline (подпись) или online через auth-сервер.
  • Жизненный цикл — проект в статусе installed переходит в «Управление объектом» как справочник, обогащается телеметрией.

Подробнее — в ROADMAP.md: Фаза 1.27 (текущая) и Фаза X (финальная, «Управление объектом как отдельный продукт»).
