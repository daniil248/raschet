# Raschet

Конструктор принципиальных схем электроснабжения. Позволяет собирать блок-схему «источники → щиты → потребители» и автоматически считает мощности с учётом АВР по приоритетам и резервных генераторов.

## Запуск

Просто откройте `index.html` в браузере. Сборка не требуется.

## Возможности

- **Палитра** (слева): источник, генератор, щит, потребитель. Перетаскиваются на холст.
- **Щит**: настраиваемое количество входов и выходов (1–3 каждого вида).
- **АВР**: для щита с несколькими входами задаются приоритеты (1 — основной). Активен тот вход, у которого запитан upstream с наивысшим приоритетом.
- **Резервный генератор**: флажок «Резервный (АВР)». Включается в работу только если ни один основной источник в цепи не доступен.
- **Расчёт мощности**: потребление каждого запитанного потребителя распространяется вверх по активным цепям. Источники показывают `потребляется / мощность`, перегруз подсвечивается красным.
- **Холст**: зум колесом, панорама по пустой области, drag-n-drop нод, связь по клику: выход → вход.
- **Сохранение**: кнопка «Сохранить» кладёт схему в localStorage; «Экспорт/Импорт JSON» — файлом.
- **Горячие клавиши**: `Del` — удалить выбранное, `Esc` — отменить ведение связи, `Shift+клик` по связи — удалить её.

## Расчётная модель (кратко)

`recalc()` строит список входящих связей для каждого узла, затем для каждого потребителя рекурсивно определяет активный фидер через `powerOf(node, allowBackup)`:

1. Сортируем входы щита по приоритету.
2. Ищем первый, у которого upstream запитан без привлечения резерва.
3. Если не нашли и `allowBackup=true` — пробуем ещё раз, разрешив резервные генераторы.
4. Распространяем `demandKw` потребителя вверх по выбранным связям, суммируя нагрузку на промежуточных щитах и конечном источнике.

Перегруз фиксируется, если суммарная нагрузка источника превышает его `capacityKw`.
